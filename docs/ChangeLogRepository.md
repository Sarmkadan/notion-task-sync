# ChangeLogRepository

Central repository for persisting and querying task change history in the Notion Task Sync system. It tracks every synchronization event between local tasks and Notion pages, enabling audit trails, conflict detection, and change statistics.

## API

### `ChangeLogRepository`
Initializes a new repository instance for managing change logs. Requires a database context and synchronization configuration to operate.

### `AddAsync`
Persists a new change log entry asynchronously. The entry is queued for storage and indexed for future queries.
- **Parameters:** `ChangeLog changeLog` – the change to record.
- **Returns:** A task representing the asynchronous operation.
- **Throws:** `ArgumentNullException` if `changeLog` is null.

### `GetByTaskIdAsync`
Retrieves all change logs associated with a specific task identifier.
- **Parameters:** `string taskId` – the unique task identifier.
- **Returns:** A list of matching change logs, ordered by timestamp descending.
- **Throws:** `ArgumentNullException` if `taskId` is null.

### `GetByDateRangeAsync`
Fetches all change logs recorded within a specified date range.
- **Parameters:**
  - `DateTime start` – inclusive start of the range.
  - `DateTime end` – inclusive end of the range.
- **Returns:** A list of change logs within the range, ordered by timestamp descending.
- **Throws:** `ArgumentException` if `start` is after `end`.

### `GetBySourceAsync`
Returns all change logs originating from a specific synchronization source.
- **Parameters:** `ChangeSource source` – the source system (e.g., Local, Notion, System).
- **Returns:** A list of change logs from the specified source, ordered by timestamp descending.

### `GetByChangeTypeAsync`
Retrieves change logs filtered by the type of change (e.g., creation, update, deletion).
- **Parameters:** `ChangeType changeType` – the type of change to filter by.
- **Returns:** A list of matching change logs, ordered by timestamp descending.

### `GetConflictChangesAsync`
Fetches all change logs marked as conflicts between local and Notion states.
- **Returns:** A list of conflict change logs, ordered by timestamp descending.

### `GetLatestAsync`
Returns the most recent change log entry across all tasks.
- **Returns:** The latest change log, or null if none exist.

### `CountAsync`
Returns the total number of change logs recorded.
- **Returns:** The count of all change logs.

### `CountConflictsAsync`
Returns the number of change logs marked as conflicts.
- **Returns:** The count of conflict change logs.

### `SaveAsync`
Persists all pending changes to the underlying store asynchronously.
- **Returns:** A task representing the asynchronous save operation.
- **Throws:** `DbUpdateException` if the underlying database operation fails.

### `GetFullAuditTrailAsync`
Retrieves the complete chronological history of all synchronization events.
- **Returns:** A list of all change logs, ordered by timestamp ascending.

### `GetStatsAsync`
Aggregates and returns high-level statistics about recorded changes.
- **Returns:** An object containing counts of total, local, Notion, system, created, updated, and conflicted changes.

### `TotalChanges`
Gets the total number of change logs recorded.
- **Type:** `int`
- **Access:** Read-only property.

### `LocalChanges`
Gets the number of change logs originating from the local system.
- **Type:** `int`
- **Access:** Read-only property.

### `NotionChanges`
Gets the number of change logs originating from Notion.
- **Type:** `int`
- **Access:** Read-only property.

### `SystemChanges`
Gets the number of change logs generated by system operations (e.g., initialization, cleanup).
- **Type:** `int`
- **Access:** Read-only property.

### `ConflictCount`
Gets the number of change logs marked as conflicts.
- **Type:** `int`
- **Access:** Read-only property.

### `CreatedCount`
Gets the number of change logs indicating new task creation.
- **Type:** `int`
- **Access:** Read-only property.

### `UpdatedCount`
Gets the number of change logs indicating task updates.
- **Type:** `int`
- **Access:** Read-only property.
