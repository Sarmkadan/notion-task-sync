# ConflictDiffService

The `ConflictDiffService` is a core utility within the `notion-task-sync` project responsible for analyzing, generating, and rendering differences between conflicting states of Notion tasks. It provides asynchronous methods to compute granular property-level diffs, aggregate batch results for multiple entities, and produce human-readable text representations of conflicts, facilitating accurate resolution strategies during synchronization operations.

## API

### `public ConflictDiffService`
Initializes a new instance of the `ConflictDiffService` class. This constructor sets up the necessary internal dependencies required to perform diff calculations and rendering operations.

### `public async Task<ConflictDiffResult> GenerateDiffAsync`
Computes a comprehensive difference result between two conflicting task states.
*   **Purpose**: Analyzes the full state of two entities to identify all discrepancies.
*   **Parameters**: Accepts the local and remote representations of the task (specific parameter types inferred from implementation context, typically source and target objects).
*   **Return Value**: Returns a `Task<ConflictDiffResult>` containing the aggregated differences, including metadata on conflict severity and affected fields.
*   **Throws**: May throw exceptions if the input objects are null, malformed, or if the underlying comparison logic encounters an invalid state transition.

### `public async Task<ConflictDiffResult> GenerateDiffForPropertyAsync`
Calculates the difference specifically for a single property within two conflicting task states.
*   **Purpose**: Isolates the comparison logic to a specific field (e.g., "Status", "Due Date") to determine if a conflict exists solely for that attribute.
*   **Parameters**: Requires the identifiers for the tasks and the specific property key or expression to evaluate.
*   **Return Value**: Returns a `Task<ConflictDiffResult>` scoped to the requested property, indicating whether values diverge and providing the respective values.
*   **Throws**: Throws if the specified property does not exist on the task schema or if the task identifiers are invalid.

### `public async Task<string> RenderAsTextAsync`
Generates a human-readable textual representation of a conflict diff.
*   **Purpose**: Converts a `ConflictDiffResult` into a formatted string suitable for logging, user interfaces, or debug output.
*   **Parameters**: Takes the `ConflictDiffResult` object generated by other methods in this service.
*   **Return Value**: Returns a `Task<string>` containing the formatted diff report.
*   **Throws**: May throw if the input result object is null or contains inconsistent data that cannot be serialized to the text format.

### `public async Task<Dictionary<Guid, ConflictDiffResult>> GenerateBatchDiffsAsync`
Efficiently computes differences for a collection of tasks identified by their unique GUIDs.
*   **Purpose**: Optimizes performance by processing multiple conflict comparisons in a single operation, reducing overhead compared to individual calls.
*   **Parameters**: Accepts a collection of task identifiers (GUIDs) and the corresponding state contexts for comparison.
*   **Return Value**: Returns a `Task<Dictionary<Guid, ConflictDiffResult>>` where the key is the Task GUID and the value is the computed diff result for that task.
*   **Throws**: Throws if any GUID in the batch cannot be resolved to a valid task state or if the batch processing pipeline fails.

## Usage

### Example 1: Resolving a Single Property Conflict
This example demonstrates how to check for a conflict on a specific property (e.g., "Status") and render the result for logging.

```csharp
var diffService = new ConflictDiffService();
var taskId = Guid.Parse("a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8");

// Generate diff specifically for the "Status" property
var propertyDiff = await diffService.GenerateDiffForPropertyAsync(
    taskId, 
    "Status", 
    localState, 
    remoteState
);

if (propertyDiff.HasConflict)
{
    // Render the conflict details for the user or log
    string report = await diffService.RenderAsTextAsync(propertyDiff);
    Console.WriteLine($"Conflict detected for task {taskId}:");
    Console.WriteLine(report);
}
```

### Example 2: Batch Processing for Sync Initialization
This example illustrates generating diffs for a list of tasks during a synchronization cycle to identify all conflicts before applying updates.

```csharp
var diffService = new ConflictDiffService();
var taskIdsToSync = new List<Guid> { /* ... list of GUIDs ... */ };

// Generate diffs for all tasks in parallel/batch
var batchResults = await diffService.GenerateBatchDiffsAsync(
    taskIdsToSync, 
    localContext, 
    remoteContext
);

foreach (var kvp in batchResults)
{
    if (kvp.Value.HasConflict)
    {
        // Handle specific conflict logic based on the result
        await ResolveConflictAsync(kvp.Key, kvp.Value);
    }
}
```

## Notes

*   **Thread Safety**: The public methods of `ConflictDiffService` are designed to be thread-safe for concurrent invocation. Multiple threads may safely call `GenerateDiffAsync` or `GenerateBatchDiffsAsync` simultaneously on the same service instance, provided the input state objects passed to these methods are not mutated externally during the operation.
*   **Null Handling**: All methods expect non-null input arguments. Passing `null` for task states, property names, or GUID collections will result in an immediate exception rather than a null result.
*   **Asynchronous Execution**: As all public members are asynchronous, callers must `await` the tasks to ensure the diff calculation completes before accessing the result. Blocking on these tasks (e.g., using `.Result`) in UI or ASP.NET contexts may lead to deadlocks.
*   **Batch Consistency**: When using `GenerateBatchDiffsAsync`, the returned dictionary contains entries only for successfully processed GUIDs. If a specific ID fails validation, the behavior depends on the configured error policy, but typically the operation may abort or skip the invalid entry while throwing for critical failures.
